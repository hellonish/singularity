# financial_search

**Tier**: retrieval  
**Name**: `financial_search`  

## Description

This skill serves as a highly specialized retrieval system designed for the precise and exhaustive extraction of financial intelligence from trusted public sources, focusing on regulatory filings and real-time news to facilitate accurate financial analysis and decision-making. Physically, it operates as an integrated retrieval pipeline that combines two key external tools: the SecEdgarTool, which interfaces directly with the U.S. Securities and Exchange Commission's (SEC) EDGAR database—a comprehensive archive containing over 20 million filings dating back to 1994—to retrieve official documents such as 10-K annual reports (detailing business operations, audited financial statements, and risk factors), 10-Q quarterly reports (covering unaudited financials and management discussions), 8-K current reports (addressing significant events like mergers, acquisitions, or executive changes), proxy statements (e.g., DEF 14A for shareholder voting and compensation details), and ownership forms (e.g., Forms 3, 4, and 5 for insider trading activities); and the WebFetchTool, which is configured with strict domain filters to gather content from reputable financial news outlets like Bloomberg.com, Reuters.com, The Wall Street Journal, Yahoo Finance, Seeking Alpha, and CNBC, targeting items such as earnings call transcripts, analyst ratings, market analyses, and breaking news on corporate developments, economic indicators, or sector trends. This dual mechanism ensures a balanced collection of structured data (e.g., XBRL-formatted financial tables from SEC filings) and unstructured data (e.g., HTML-based news articles), executed through secure API queries or compliant web scraping that adheres to robots.txt files, user-agent headers, and legal access protocols to prevent unauthorized scraping or IP blocking.

Cognitively, the skill mimics the methodical workflow of a seasoned financial analyst, employing a multi-layered reasoning process that begins with advanced natural language processing (NLP) on the input query. This starts with lexical analysis, including tokenization to decompose the query into individual elements, followed by named entity recognition (NER) using state-of-the-art models like spaCy or fine-tuned BERT variants trained on financial datasets to identify and categorize entities such as stock tickers (e.g., mapping "Apple" to "AAPL" via cross-referencing with authoritative databases like NASDAQ or NYSE), date ranges (e.g., interpreting "last fiscal year" as "2023-01-01 to 2023-12-31" using libraries like dateutil for precise temporal resolution), document types (e.g., recognizing "quarterly earnings" as linked to 10-Q forms), quantitative metrics (e.g., "revenue growth rate" or "earnings per share"), and thematic concepts (e.g., "market volatility" or "regulatory compliance"). Semantic enrichment follows, involving dependency parsing to uncover relationships between query components and intent classification via vector embeddings (e.g., Sentence Transformers) to calculate query-document similarity scores, thereby resolving ambiguities (e.g., differentiating "bear market" from unrelated homonyms) and generating refined search vectors. This leads to query reformulation: for SecEdgarTool, it constructs targeted EDGAR queries using parameters like CIK codes, accession numbers, filing dates, and form types (e.g., filtering for all 10-K filings of a specific ticker within a date range); for WebFetchTool, it formulates precise search strings or URL patterns (e.g., "site:reuters.com AAPL earnings 2023" with temporal filters). The cognitive process is adaptive, prioritizing factors like recency (e.g., defaulting to filings from the last five years unless overridden), relevance (e.g., applying thresholds for keyword matches and semantic alignment), and source credibility, ensuring searches are comprehensive yet efficient.

In terms of specific data transformations, the skill applies a sophisticated pipeline to convert raw, heterogeneous inputs into a standardized, actionable dataset suitable for downstream use. Upon retrieval, raw data in formats like PDF (from SEC filings), HTML or JSON (from news APIs), or XBRL (for structured financial data) undergoes initial preprocessing: for PDFs, optical character recognition (OCR) via tools like Tesseract is applied to digitize scanned documents, followed by layout parsing using libraries such as PyMuPDF to extract key sections (e.g., isolating balance sheets, income statements, or MD&A narratives). HTML content is parsed with BeautifulSoup to strip away noise like advertisements and focus on core text. Extraction then employs a hybrid approach: rule-based methods (e.g., regex patterns to identify tickers like [A-Z]{1,5} or dates in ISO formats) combined with machine learning techniques (e.g., fine-tuned transformers for entity extraction) to pull out essential metadata, including ticker symbols, form types, fiscal periods, filing dates, source URLs, and excerpt snippets (e.g., the first 500 characters of relevant content). Transformations include deduplication (e.g., using content hashing to remove duplicates across sources), normalization (e.g., converting all dates to UTC and currencies to USD for consistency), and augmentation (e.g., assigning credibility scores based on source type: 1.0 for direct SEC filings, 0.90 for verified transcripts, and 0.80 for established news outlets, calculated via a formula weighing factors like source age, citation volume, and editorial reliability). The output is aggregated into a ranked list, limited to a maximum of 10 items based on relevance scoring (e.g., BM25 for keyword matching plus cosine similarity on embeddings), and formatted as a JSON array with embedded provenance details (e.g., {"sources": [{"ticker": "AAPL", "form_type": "10-K", "fiscal_period": "2023", "filing_date": "2024-01-15", "excerpt": "Revenue increased by 10%...", "source_url": "https://www.sec.gov/..."}]}). This transformation ensures the data is verifiable, concise, and free from generative alterations, maintaining factual integrity by focusing solely on extracted content.

## When to Use

This skill should be deployed in precise scenarios where authoritative, verifiable financial data is essential for analysis, compliance, or strategic decision-making, ensuring that retrieved information from primary sources drives accurate insights without reliance on secondary or interpretive content. Specific scenarios include conducting due diligence for investments, such as fetching a company's complete set of 10-K and 10-Q filings to evaluate financial health, risk exposures, and operational metrics (e.g., analyzing revenue breakdowns or debt covenants for a potential acquisition); supporting earnings analysis during reporting seasons by retrieving Q4 transcripts and associated news from Reuters to compare guidance with actual results and assess market reactions; facilitating sector-wide research, like compiling 8-K filings and Bloomberg articles on regulatory investigations (e.g., antitrust issues in technology sectors) to benchmark company performance; tracking macroeconomic trends by pulling SEC filings and news updates on interest rate changes for major indices like the S&P 500; or performing forensic reviews, such as cross-referencing Form 4 insider trading disclosures with WSJ reports to detect patterns in executive stock sales. It is particularly valuable in compliance-driven contexts, such as verifying ESG claims in proxy statements for sustainability audits or assembling historical news archives for volatility forecasting in portfolio management.

Upstream dependencies are critical for optimal performance: the skill requires inputs in the form of a well-structured string query (e.g., "Fetch AAPL 10-K for fiscal year 2023 and related earnings news from Bloomberg") or a JSON object (e.g., {"query": "AAPL", "form_types": ["10-K", "10-Q"], "date_range": {"start": "2023-01-01", "end": "2024-01-01"}, "keywords": ["revenue", "growth"]}), which must be pre-processed by prior nodes, such as a query parser agent that resolves ambiguities (e.g., converting "Apple's latest reports" to a standardized {"ticker": "AAPL", "form_types": ["10-K"]}) or an entity extraction agent that enriches inputs with contextual details (e.g., appending jurisdiction filters like "US-only" or expanding acronyms). These upstream steps ensure inputs are entity-rich, parameterized, and exceed a minimum threshold (e.g., including at least one ticker, date range, or keyword to avoid vague queries), as unrefined inputs (e.g., "info on tech stocks") could lead to inefficient or irrelevant results.

Edge cases where this skill should not be used include situations involving non-financial topics, such as general news on politics, social trends, or scientific advancements, where broader web search tools or specialized databases (e.g., academic repositories) would be more appropriate; real-time market data needs, like current stock prices, options quotes, or live trade executions, as the skill's sources have inherent delays (e.g., SEC filings are posted 24-48 hours after events, and news updates may lag by minutes), potentially causing outdated or misleading information in high-frequency trading scenarios; queries restricted by access issues, such as API downtimes, geo-blocks on EDGAR (e.g., for non-US users without proper routing), or attempts to access private data (e.g., unpublished venture capital deals), which could result in empty outputs or errors; ambiguous or overly broad queries (e.g., "good news on stocks" without parameters), which might generate noisy results—direct such cases to clarification agents instead; and low-scale tasks, like simple fact-checking of a single metric, where cached data sources or lightweight tools suffice, or international financial data (e.g., EU filings under EMIR), which require dedicated global retrievers due to this skill's U.S.-centric focus.

Downstream nodes that typically follow include a financial summarizer to process outputs and extract key performance indicators (e.g., parsing EBITDA from 10-K tables using regex or NLP), a sentiment analyzer to evaluate news excerpts for market polarity (e.g., applying VADER on Reuters text to score sentiment scores), quantitative analyzers that integrate metadata into data frames for statistical modeling (e.g., calculating financial ratios in pandas), or report generators that compile sources into structured narratives with citations (e.g., creating investment briefs). In a DAG, this skill often precedes validation nodes (e.g., date verification against external calendars) and branches to specialized agents (e.g., risk assessment for 8-K events or trend forecasting), ensuring the retrieved data feeds into iterative analysis without redundancy.
